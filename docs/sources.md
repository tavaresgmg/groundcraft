# Sources and lineage

Groundcraft is an original synthesis. It adopts ideas, not copied prompt text.

## Agent methods and tools

- [Matt Pocock Skills](https://github.com/mattpocock/skills): small composable skills, triggering, progressive disclosure, fresh-context tickets, and completion criteria. MIT.
- [GSD Core](https://github.com/open-gsd/gsd-core): context engineering, durable state, phase loops, and fresh-context execution. MIT.
- [BMAD Method](https://github.com/bmad-code-org/BMAD-METHOD): just-in-time files, explicit contracts, adversarial review, and halt states. MIT.
- [Superpowers](https://github.com/obra/superpowers): design gates, test-first practice, independent review, and evidence before completion. MIT.
- [Ponytail](https://github.com/DietrichGebert/ponytail): YAGNI, reuse, standard-library and native-first design, and complexity review. MIT.
- [gstack](https://github.com/garrytan/gstack): browser QA, role-separated review, release discipline, and a caution against prompt duplication. MIT.
- [OpenSpec](https://github.com/Fission-AI/OpenSpec): lightweight brownfield change agreements and delta specifications. MIT.
- [GitHub Spec Kit](https://github.com/github/spec-kit): constitutions, executable planning artifacts, and workflow gates. MIT.
- [Nanostack](https://github.com/garagon/nanostack): capability honesty, artifact-first coordination, and local orchestration. Apache-2.0.

## Engineering evidence

- [DORA 2025](https://dora.dev/research/2025/dora-report/): AI amplifies the surrounding delivery system.
- [Working in small batches](https://dora.dev/capabilities/working-in-small-batches/) and [trunk-based development](https://dora.dev/capabilities/trunk-based-development/): feedback and integration practices.
- [Google Engineering Practices](https://google.github.io/eng-practices/): small changes, review standards, tests, and review communication.
- [Google SRE: Canarying Releases](https://sre.google/workbook/canarying-releases/): controlled rollout and operational evidence.
- [Parnas, 1972](https://doi.org/10.1145/361598.361623): information hiding and module boundaries.
- [NIST SSDF](https://csrc.nist.gov/projects/ssdf) and [SLSA](https://slsa.dev/spec/v1.2/): secure development and artifact provenance.
- [NIST AI RMF](https://airc.nist.gov/airmf-resources/airmf/5-sec-core/): risk-aware human oversight.
- [Anthropic agent evals](https://www.anthropic.com/engineering/demystifying-evals-for-ai-agents): capability and regression evaluation design.
- [OpenAI on SWE-bench Verified](https://openai.com/index/why-we-no-longer-evaluate-swe-bench-verified/): benchmark contamination and grader quality limits.

Community discussions were used to discover failure modes such as context bloat, command fatigue, document drift, unnecessary questions, and self-review bias. Normative technical claims remain grounded in primary sources.
